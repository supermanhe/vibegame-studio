# VibeGame Studio: 游戏研发革命

这是一份基于深度讨论整理的**项目企划书**。它将你的直觉（Vibe）转化为了一个逻辑严密的商业方案，旨在解决游戏行业最核心的研发痛点。

---

## 项目名称：VibeGame Studio (暂定)

**核心口号：** "在开启 Unity/Unreal 之前，先在云端踩完所有的坑。"

---

## 一、项目背景与痛点 (The Problem)

1. **研发成本极高：** 传统游戏开发"先造车再试驾"，一旦原型不好玩，数月的研发经费和人力瞬间打水漂。

2. **"纸上谈兵"的策划案：** 动辄几十页的 Word 文档（GDD）逻辑不透明、数值不平衡，程序和美术难以理解，导致大量返工。

3. **数值黑盒：** 手游/商业游戏极其依赖数值，但由于缺乏大规模模拟，上线后经常出现数值崩盘或节奏断层。

4. **沟通鸿沟：** 策划的"主观感受"与程序的"客观实现"之间存在巨大的翻译成本。

---

## 二、产品定位 (Positioning)

**全球首款 AI 驱动的"游戏设计验证平台 (Game Design Validation Platform)"。**

- **属性：** 专业级生产力工具（B2B / 专业向）
- **目标：** 帮助制作人、策划在 10 分钟内将模糊想法转化为**逻辑闭环**、**数值平衡**、**可交互**的白盒原型

---

## 三、核心功能模块 (Key Features)

### 1. 多代理虚拟工作室 (Agentic Studio)

内置多个垂直领域的 AI 专家，协同完成任务：

| 角色 | 职责 |
|------|------|
| **主制作人** | 提炼核心玩法支柱（Pillars） |
| **系统策划** | 构建用户流程、UI结构、信息架构 |
| **战斗策划** | 构建行为树、关卡逻辑、状态机 |
| **数值策划** | 编写 Python 脚本进行 10 万次模拟测试，确保数值不崩盘 |
| **主策划 (Critic)** | 负责 Review 各环节产出，不达标自动触发循环重写 |

### 2. 白盒可交互原型 (White-box Interactive Demo)

- **逻辑验证：** 自动生成基于网页（Three.js/React）的 3D/2D 白盒原型
- **即时体验：** 策划和老板可以直接在网页里测试手感、关卡节奏和解谜逻辑
- **实时微调：** 用户通过对话或简单的 2D 地图编辑，实时修改原型逻辑，AI 自动重构代码

### 3. 工业级生产力导出 (Hardcore Exports)

不只是输出文字，而是直接输出可集成的资产：

| 导出类型 | 说明 |
|----------|------|
| **数值资产** | 生产环境可用的 JSON/CSV 数值表 |
| **逻辑架构** | 自动生成 C# / C++ 状态机框架代码 |
| **视觉逻辑** | 自动生成 Mermaid 流程图和行为树结构 |
| **可交互原型** | 自动生成基于网页（Three.js/React）的 3D/2D 白盒原型 |
| **动作帧表** | 量化攻击前摇、判定、后摇的毫秒级数据 |

## 四、核心工作流 (Workflow)

```
1. 概念输入 → 2. 知识注入 → 3. 自动化循环 → 4. 原型交付 → 5. 接驳生产
```

1. **概念输入：** 用户输入模糊的玩法想法（Vibe）
2. **知识注入：** Agent 检索内置的"优秀策划案知识库"，学习顶级设计标准
3. **自动化循环：** Agent 在后台进行"计划-执行-模拟-评价"的闭环迭代
4. **原型交付：** 输出一个可玩的网页链接，供团队体验
5. **接驳生产：** 确认方案后，一键导出数据和代码框架，无缝接入 Unity/Unreal

## 五、行业价值 (Value Proposition)

| 角色 | 价值 |
|------|------|
| **对制作人** | 将试错成本从"几十万"降到"几块钱"，极大提高立项成功率 |
| **对策划** | 告别被程序怼"逻辑不通"，用"活的说明书"证明玩法的可行性 |
| **对程序** | 接收到的是经过验证的逻辑和数值，不再有模糊需求，不再有无效加班 |
