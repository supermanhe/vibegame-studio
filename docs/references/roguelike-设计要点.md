# Roguelike 游戏设计要点

> 这是一个示例参考文档，展示知识库文档的格式。
> 你可以在这个目录下添加更多参考资料。

## 核心特征

### 1. 永久死亡 (Permadeath)
- 角色死亡后失去所有进度
- 增加紧张感和决策权重
- 每次游戏都是独特体验

### 2. 随机生成 (Procedural Generation)
- 关卡/地图随机生成
- 每次游戏都有新发现
- 提高重玩价值

### 3. 回合制 (Turn-based)
- 传统 Roguelike 采用回合制
- 现代变体（Roguelite）可以是实时
- 给玩家思考时间

### 4. 资源管理
- 有限的回复资源
- 物品选择的取舍
- 风险与收益的平衡

## 优秀设计案例

### Hades
- **成功要素**：叙事与 Roguelike 结合
- **永久升级**：暗夜之镜系统
- **剧情推进**：死亡不是惩罚，而是推进剧情的方式

### Slay the Spire
- **成功要素**：卡牌构筑 + Roguelike
- **核心循环**：战斗 → 选牌 → 构筑 → 更强战斗
- **深度策略**：卡组协同效应

### Dead Cells
- **成功要素**：动作 + Roguelike
- **手感优先**：流畅的战斗操作
- **永久升级**：解锁新武器和能力

## 设计注意事项

### 随机性控制
- 避免纯运气决定输赢
- 提供选择空间
- 让玩家感觉"我本可以做得更好"

### 新手引导
- Roguelike 学习曲线陡峭
- 需要良好的新手教程
- 渐进式解锁复杂度

### 游戏时长
- 单局时长要合适（10-60分钟）
- 手游版本需要更短
- 支持暂停/存档
